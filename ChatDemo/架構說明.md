# 專案分層架構說明

本專案採用 .NET 8 + Vue 前後端分離，並依企業級實務進行五層分層設計，方便維護、擴充與團隊協作。

---

## 一、分層架構總覽

```
ChatDemo/
├── Contracts/         # 介面層，定義服務介面、DTO、共用屬性
├── Application/       # 業務邏輯層，實作服務邏輯，僅依賴 Contracts
├── Implement/         # 實作層，連接資料庫/外部服務，實作 Application 介面
├── Models/            # 資料結構層，定義資料模型
├── Api/Controllers/   # 控制器層，對外 API 入口
└── ...
```

---

## 二、各層職責說明

- **Contracts**：
  - 定義所有服務介面（如 IMemberService）、DTO、共用屬性（Attribute）。
  - 僅描述「要做什麼」，不含實作細節。
  - 範例：
    ```csharp
    // Contracts/Member/IMemberService.cs
    /// <summary>
    /// 會員服務介面
    /// </summary>
    public interface IMemberService {
        MemberDto GetMemberById(int id); // 取得會員資料
    }
    ```

- **Application**：
  - 負責業務邏輯實作，僅依賴 Contracts。
  - 不直接操作資料庫。
  - 範例：
    ```csharp
    // Application/Member/MemberService.cs
    /// <summary>
    /// 會員業務邏輯
    /// </summary>
    public class MemberService : IMemberService {
        // ... 實作細節 ...
    }
    ```

- **Implement**：
  - 連接資料庫、外部 API，實作 Application 介面。
  - 可有多種實作（如真實/Mock）。
  - 範例：
    ```csharp
    // Implement/Member/MemberServiceImpl.cs
    /// <summary>
    /// 會員服務實作（資料庫）
    /// </summary>
    public class MemberServiceImpl : IMemberService {
        // ... 資料庫存取 ...
    }
    ```

- **Models**：
  - 定義資料結構（如 Member、Order）。
  - 範例：
    ```csharp
    // Models/Member.cs
    /// <summary>
    /// 會員資料模型
    /// </summary>
    public class Member {
        public int Id { get; set; }
        public string Name { get; set; }
    }
    ```

- **Api/Controllers**：
  - 對外 API 入口，呼叫 Application 層服務。
  - 範例：
    ```csharp
    // Api/Controllers/MemberController.cs
    /// <summary>
    /// 會員 API 控制器
    /// </summary>
    [ApiController]
    [Route("api/[controller]")]
    public class MemberController : ControllerBase {
        private readonly IMemberService _memberService;
        // ... 建構子注入 ...
    }
    ```

## 三、Mock 切換說明

- **Program.cs** 內設有 `useMock` 變數，可一鍵切換 Mock/真實服務。
- Mock 服務可用於前端開發、測試，不需連接資料庫。
- 範例：
    ```csharp
    // Program.cs
    bool useMock = true; // 切換為 Mock
    if (useMock) {
        services.AddScoped<IMemberService, MockMemberService>();
    } else {
        services.AddScoped<IMemberService, MemberServiceImpl>();
    }
    ```

## 四、資料夾細分建議

- **依功能/模組（SID）細分**：每層下可再依 Member、Order、Product 等模組分資料夾。
- **Utility**：放置共用工具類（如加解密、格式化）。
- **Attribute**：放置自訂屬性（如驗證、日誌）。
- **好處**：
  - 模組化、易維護、團隊分工明確。
  - 共用邏輯集中管理，減少重複。

## 五、常見問題指引

1. **dotnet restore 作用？**
   - 還原 .NET 專案所需 NuGet 套件。
2. **如何執行 sh 檔？**
   - `chmod +x xxx.sh` → `./xxx.sh`
3. **npm run dev 找不到 script？**
   - 請確認 chatapp/package.json 內有對應 script，並於 chatapp 目錄下執行。
4. **Browserslist 警告？**
   - 執行 `npx update-browserslist-db@latest` 更新。
5. **本專案是微服務嗎？**
   - 不是，為單一應用分層設計。
6. **分層內部可否再細分？**
   - 建議依功能/模組細分，提升可維護性。
